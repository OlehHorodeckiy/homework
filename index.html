Шпаргалка по командам Git

  • git config --global user.name blabla- наше ім'я які будуть видимі при коміті в репозиторій
  • git config --global user.email blabal@bla - пошта, яеп буде видимі при коміті в репозиторію
  • git init - ніціює пустий репозиторій в поточній директорії. Так само можна вказати будь-який шлях.
  • git status -відображає стан робочого каталогу та розроблених файлів.
  • git add . – додаємо файли в коміт. Зазвичай використовується «.» або «*» для додавання усіх файлів. Щоб додати один файл – вказуємо його ім'я та розширення.
  • git commit -"Повідомлення, яке описує комміт" – За допомогою цієї команди ми виконуємо «коміт» змін. Тобто, вказуємо, що поточний зліпок проекту потрібно зберегти.
  • git log перечисляє комміти, укладені в репозиторії в обратному хронологічному порядку - останні комміти знаходяться вверху.
  • git checkout - файли копіюють файли со сценами в робочий каталог. Ету команду зручно використовувати, щоб переглядати небажані зміни в робочих директоріях.
  • Команда git branch - це свого рода "менеджер веток". Вона вміє перечисляти ваші вітки, створювати нові, віддаляти і переіменовувати їх.
  • git push origin master – Пушим» або відправляємо в репозиторій наші зміпки проекту.
  • git pull origin master – команда для отримання змін з репозиторію.
  • git merge використовується для злиття однієї або кількох векторів у поточній.
  • git clone - це утиліта командної строки Git для вибору існуючого репозиторію та створення його клону, т. е. копії.

gitignore необхідний для приховування файлів та папок від системи контролю версії Git. Звичайно приховують конфігураційні файли (особливо з паролями), тимчасові файли та папки. gitignore використовує глобальний формат для вибору файлів.
